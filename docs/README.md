
# Nutrition Tracker Backend API

The current project is a mock backend API for an AI assisted meal and nutrition tracking web application based on Node.js, so Node.js should be installed in order to run this program.

Node.js can be installed via the official Node.js website or with a package manager like Homebrew for macOS.

The idea of the final project is a system that helps users document meals and track daily nutrition values. In the final version, the user will be able to upload a meal image, get AI based food suggestions, edit the result, confirm the meal, and then see daily progress against nutrition goals.

In this assignment, the backend uses mock data only. There is no real database connection and no real AI API call.

## How to install

From the project root folder, run:

```bash
npm install
```

This installs the dependencies from `package.json`, including Express.

There is no need to run `npm install express` separately because Express is already listed in the project dependencies.

## Important assumptions

- Data is stored only in memory.
- Data resets when the server restarts.
- No real database is connected in this assignment.
- No real AI model is called in this assignment.
- The AI image analysis endpoint returns a mocked result.
- Authorization is simulated with the `x-user-role` request header.
- IDs are generated by taking the current maximum ID and adding 1.
- The dashboard goals are currently hardcoded mock values.

---

## Response format

All successful responses follow this format:

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

All error responses follow this format:

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "SOME_ERROR_CODE",
    "message": "Human readable message",
    "details": {}
  }
}
```

---

## Authorization

Some routes are protected by a role-based middleware.

The role is sent using this header:

```txt
x-user-role: admin
```

Supported roles:

```txt
admin
manager
user
```

Role permissions:

| Action | Allowed roles |
|---|---|
| Create user | `admin`, `manager` |
| Update user | `admin`, `manager` |
| Delete user | `admin` |
| Create meal | `admin`, `manager`, `user` |
| Update meal | `admin`, `manager`, `user` |
| Delete meal | `admin` |
| Analyze meal image | `admin`, `manager`, `user` |

If the role is missing or not allowed, the API returns `403 Forbidden` in the required error response format.

# API Reference

## Endpoint summary

| Method | Path | Description | Body / params |
|---|---|---|---|
| GET | `/` | Check that the API is running | none |
| GET | `/users` | Get all users | none |
| GET | `/users/:id` | Get one user by ID | path param: `id` |
| POST | `/users` | Create a user | JSON body: `firstName`, `lastName`, `userRole` |
| PUT | `/users/:id` | Update a user | path param: `id`, JSON body: `firstName`, `lastName`, `userRole` |
| DELETE | `/users/:id` | Delete a user | path param: `id` |
| GET | `/meals` | Get all meals | optional query params: `userId`, `date` |
| GET | `/meals/:id` | Get one meal by ID | path param: `id` |
| POST | `/meals` | Create a meal | JSON body: `userId`, `mealName`, `mealDate`, `items` |
| PUT | `/meals/:id` | Update a meal | path param: `id`, JSON body: `userId`, `mealName`, `mealDate`, `items` |
| DELETE | `/meals/:id` | Delete a meal | path param: `id` |
| POST | `/meals/analyze-image` | Mock AI meal image analysis | JSON body: `imageName` |
| GET | `/dashboard/today` | Get daily nutrition dashboard | query param: `userId`, optional `date` |

## Request body formats

### Create / update user

Used by:

```txt
POST /users
PUT /users/:id
```

```json
{
  "firstName": "New",
  "lastName": "User",
  "userRole": "user"
}
```

### Create / update meal

Used by:

```txt
POST /meals
PUT /meals/:id
```

```json
{
  "userId": 1,
  "mealName": "manual tuna sandwich",
  "mealDate": "2026-05-06",
  "imagePath": null,
  "items": [
    {
      "foodName": "tuna",
      "confirmedPortionGrams": 120,
      "calories": 140,
      "protein": 30,
      "carbs": 0,
      "fat": 1
    }
  ]
}
```

### Mock image analysis

Used by:

```txt
POST /meals/analyze-image
```

```json
{
  "imageName": "lunch.jpg"
}
```

The value `lunch.jpg` is only a mock file name. This assignment does not upload a real image file.

## Common status codes

| Status code | Meaning |
|---|---|
| `200 OK` | Successful GET, update, or delete |
| `201 Created` | Successful POST that creates a new item |
| `400 Bad Request` | Validation error, missing required field, invalid ID, or invalid JSON |
| `403 Forbidden` | Missing or invalid role for a protected route |
| `404 Not Found` | Route or requested item does not exist |
| `500 Internal Server Error` | Unexpected server error |
