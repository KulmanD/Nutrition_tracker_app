
# Nutrition Tracker Backend API

This project is a mock backend API for an AI-assisted meal and nutrition tracking web application.

The idea of the final project is a system that helps users document meals and track daily nutrition values. In the final version, the user will be able to upload a meal image, get AI-based food suggestions, edit the result, confirm the meal, and then see daily progress against nutrition goals.

For this assignment, the backend is intentionally simple. It does **not** use MySQL yet, and it does **not** call a real AI API. The goal here is to build a clean Express API skeleton with mock data, so later we can replace the mock data with a real database without changing the whole API design.

---

## Project status for Assignment 2

This backend includes:

- Node.js + Express server
- Mock data stored in memory
- Users CRUD
- Meals CRUD
- Mock AI meal image analysis endpoint
- Daily dashboard endpoint
- Basic validation
- Global logger middleware
- Role-based authorization middleware
- Consistent JSON success and error responses
- Postman collection for testing

This is not the final full system yet. It is the backend API skeleton for the assignment milestone.

## Important assumptions

- Data is stored only in memory.
- Data resets when the server restarts.
- No real database is connected in this assignment.
- No real AI model is called in this assignment.
- The AI endpoint returns a mocked result.
- Authorization is simulated with the `x-user-role` request header.
- IDs are generated by taking the current maximum ID and adding 1.
- The dashboard goals are currently hardcoded mock values.

These assumptions are intentional because Assignment 2 focuses on the backend API skeleton, not the final database or AI implementation.

## Response format

All API responses follow the same general structure.

### Success response

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

### Error response

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "SOME_ERROR_CODE",
    "message": "Human readable message",
    "details": {}
  }
}
```

This makes the API easier to test because the client always knows where to find the result and where to find the error.

## Authorization

Some routes are protected by a simple role-based middleware.

The role is sent using this header:

```txt
x-user-role: admin
```

Supported roles:

```txt
admin
manager
user
```

### Current role permissions

| Action | Allowed roles |
|---|---|
| Create user | `admin`, `manager` |
| Update user | `admin`, `manager` |
| Delete user | `admin` |
| Create meal | `admin`, `manager`, `user` |
| Update meal | `admin`, `manager`, `user` |
| Delete meal | `admin` |
| Analyze meal image | `admin`, `manager`, `user` |

If the role is missing or not allowed, the API returns `403 Forbidden` in the required error response format.

Example:

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "FORBIDDEN",
    "message": "Missing user role. Please send x-user-role header.",
    "details": {
      "requiredHeader": "x-user-role",
      "allowedRoles": ["admin", "manager", "user"]
    }
  }
}
```

## Logger middleware

The server includes a global logger middleware.

It logs basic information for every request:

- date and time
- HTTP method
- requested URL
- response status code
- time taken

Example server log:

```txt
[2026-05-06T15:16:11.312Z] GET /users - 200 - 1ms
```

This helps during development because we can see which endpoints were called and what status they returned.

# How this connects to the final project

This backend is designed so that later we can replace the mock parts with real parts:

- Replace `models/usersData.js` with real users table queries.
- Replace `models/mealsData.js` with real meals and meal items tables.
- Replace the mock `aiController.js` result with a real AI/image analysis API call.
- Replace hardcoded dashboard goals with real user nutrition goals from the database.

The main API contract should stay mostly the same, which is the point of building the skeleton now.
